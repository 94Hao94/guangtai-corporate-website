# 天津光泰科技集团官网说明文档

## 项目概述

本项目是天津光泰科技集团官网的 Astro 静态站点，承载 AI 应用工厂、解决方案、行业场景、案例与客户、关于光泰和联系入口。当前动效改造工作分支为 `feat/ai-motion-navigation`，用户预览地址为 `http://127.0.0.1:4324`。

## 技术栈

- Astro 7 和 TypeScript 6
- Tailwind CSS 4 与项目内 CSS 样式系统
- Astro Content Collections 管理页面 JSON 内容
- Vitest 4 负责内容与静态输出测试
- Playwright 1.61 负责浏览器端到端测试
- GSAP 3 与 ScrollTrigger 负责可清理的页面进入动效
- Astro ClientRouter 负责同源页面的无刷新切换
- Docker 和非 root Nginx 用于静态部署

## 问题与解决方案

- 问题：`/cases` 首屏显示带资产名称的内部拼图，损害专业感。
  解决：保留现有版式，只替换 Hero 与 Open Graph 图片，并加入内容合同测试防回归。
- 问题：Astro 开发服务未自动刷新内容集合，修改后仍提供旧图。
  解决：重启当前 worktree 的开发服务并绑定 `127.0.0.1:4328`，确认页面请求新图。
- 问题：迁移前 MVP 输入目录不再存在，遗留合同测试导致全量 Vitest 失败。
  解决：按当前确认的验证范围移除该测试，保留迁移后内容、构建和浏览器测试。
- 问题：无刷新进入内页后，鼠标落点会先触发解决方案菜单的悬停展开，随后同一次点击又将菜单关闭，造成“点击没有变化”的感受。
  解决：将首次点击定义为锁定打开，第二次点击关闭；持久化 Header 仅初始化一次交互监听，并在 `astro:after-swap` 同步当前方案状态。
- 问题：通过 ClientRouter 离开首页再返回时，首屏数据网络动画只播放一次，新画布会保留默认的 `300` 像素宽度。
  解决：在 `astro:page-load` 生命周期中清理旧动画循环，并为当前路由重新查找并初始化 `#ai-canvas`；监听器只注册一次，避免重复绑定。
- 问题：首页以外的动效初始化只写在首页模块脚本中，ClientRouter 切换后合作伙伴轮播停止、内页没有统一的滚动进入动效；菜单又误缩为 4 个总入口，且指针斜向进入时立即关闭。
  解决：由 `BaseLayout` 在每次 `astro:page-load` 重建并清理 GSAP context；页面组件只提供 `data-reveal` 锚点。菜单按现有路由恢复四组 24 项，使用可取消的 220ms 关闭计时器，并改为半透明深蓝玻璃面板。
- 问题：菜单点击外部关闭逻辑将整个持久化 Header 视作“菜单内部”，点击其他头部导航后菜单状态会带入下一页。
  解决：将判定范围收窄为解决方案触发器与面板本身；点击任意其他目标都会关闭方案菜单，同时保留移动导航仅在 Header 外点击时关闭的规则。
- 问题：首页 Hero 同时呈现产品标题、长段卖点、双 CTA、能力标签和状态标识，营销信息过密，弱化企业品牌气质。
  解决：保留已有科技背景与数据网络，替换为单行品牌主张、公司全称和单一“了解更多”按钮；按钮指向“关于光泰”页面。
- 问题：收敛内容后的 Hero 文字组仍沿用原先的顶部 padding，在不同视口中偏离视觉中心。
  解决：让文字容器继承 Hero 高度并采用垂直居中布局，以底部留白微调至约 46.6% 的视觉重心；通过浏览器断言保证其保持在 Hero 高度的 42% 至 60% 区间。
- 问题：只有 slogan 与公司名称时，首屏对 AI 应用工厂的实际能力表达不足。
  解决：在 slogan 下加入“以AI应用工厂为平台底座，融合软件开发、系统集成、空间智能与具身设备。”，使用编号、细边线和较小字号组成次级信息块，不削弱品牌主张。
- 问题：静态 Hero 机器人缺少互动感，不能体现具身智能业务的科技感。
  解决：使用用户提供的 Spline 公开场景，通过官方 `@splinetool/viewer` 在桌面端按需挂载；场景加载成功后淡出原静态图，移动端或减少动态效果偏好继续显示静态回退。外部 WebGL 场景的最终交互手感需在用户浏览器中验收。
- 问题：为收敛旧场景构图而给 Spline Viewer 添加 CSS 缩放与位移后，Viewer 的实际画布坐标与鼠标位置不再一致，导致跟随交互只在局部区域生效。
  解决：移除所有 Viewer CSS 变形，保留完整 80vw 交互 Canvas；使用用户重新导出的场景本身完成机器人尺寸与脚线构图，并追加 `?v=robot-v2` 使浏览器获取新导出版本。
- 问题：透明的 Hero 文字层位于 3D Viewer 上方，即使没有可见背景仍会截获指针事件，使机器人只能在未覆盖的右侧区域响应鼠标。
  解决：文字层默认 `pointer-events: none`，仅“了解更多”链接恢复点击；同时通过正常布局将 80vw Canvas 整体右移至机器人目标区域，避免 CSS transform 造成新的坐标错位。
- 问题：Playwright 全页完整性测试会逐张滚动并等待惰性图片加载；页面资源增多后超过默认 30 秒，但每次超时卡在不同图片，证明不是固定资源失效。四视口截图测试又重复了同一遍历四次，偏离其截图职责。
  解决：不修改页面图片或交互，只为独立完整性检查增加长时预算；四视口测试收敛为可见性、横向溢出与截图验证，完整性仍由前者覆盖。

## 部署说明

用户预览使用 `npm run dev -- --host 127.0.0.1 --port 4324`。完整静态验证使用 `npm run verify`；在受限桌面环境中，Playwright Chromium 需要允许正常的本地浏览器进程权限。生产预览使用 Docker Compose 的 8080 端口，构建时必须设置 `SITE_URL`。
