# 首页 AI 能力优先级调整设计

## 背景

首页首屏已使用可交互机器人，能够体现光泰具身智能业务。当前首个内容区仍是具身智能，AI 应用工厂位于更靠后的区块，容易让访问者将公司理解为只提供机器人相关业务。实际业务还包括 AI 开发、行业智能体、知识库与模型工程、软件交付及私有化部署。

用户要求提升 AI 业务在首页中的优先级，同时保持现有首页版式、素材、卡片、合作伙伴轮播和动效的视觉设计，不进行重构式改版。

## 目标

- 首屏后的第一个业务章节明确展示 AI 应用工厂与行业智能体能力。
- AI 后紧接具身智能，并让具身智能合作伙伴继续紧跟该章节，形成完整的生态证明。
- 在具身智能及其合作伙伴之后，呈现传统系统集成与行业场景能力。
- 保留所有现有首页区块、链接、图片、响应式规则和动效锚点；仅调整顺序，并对重复或不够准确的业务表述做最小收敛。

## 非目标

- 不修改首页 Hero 的机器人、Spline 场景、首屏构图或交互方式。
- 不更换首页图片、合作伙伴 Logo、案例、卡片布局或色彩系统。
- 不新增路由、不改变既有 CTA 目标，也不调整站点头部、菜单或内页。
- 不重构现有 CSS 架构或动效生命周期。

## 信息架构

首页业务叙事调整为：

`HomeHero → FactorySection → EmbodiedSection → PartnerMarquee → SolutionSplit → EducationSection → TechnologySection → CaseSection → ProjectCta`

各章节的角色如下：

1. `FactorySection`：AI 应用工厂、AI 开发与行业智能体的能力入口。保留现有工厂架构图和四层能力列表，作为首个内容区。
2. `EmbodiedSection`：具身智能的设备与真实空间执行能力。保持现有图片、列表和版式。
3. `PartnerMarquee`：具身智能相关合作伙伴生态证明。必须紧随具身智能章节，不可后置到传统集成或案例之后。
4. `SolutionSplit`：传统系统集成与行业场景的承接章节。保留现有双栏大图版式与两个既有链接；标题、导语和第一张卡片文案收敛为系统集成、设备接入和工程交付，避免与前面的 AI、具身内容重复竞争。
5. 其余教育、技术工程、案例和咨询区块保持原顺序及设计。

## 内容调整

- `FactorySection` 的现有标题“把模型、知识、工具和流程编排成可运行的行业智能体”保留；必要时只补强其 AI 开发与交付语义，不改变排版层级。
- `EmbodiedSection` 与 `PartnerMarquee` 不改内容和视觉表现。
- `SolutionSplit` 的章节标题改为强调“传统系统集成与行业落地”；导语说明其承接空间、设备、基础设施、音视频、安全治理与软件集成。
- 第一张解决方案卡片以“系统集成与工程交付”为主要认知，仍链接 `/solutions/common`；第二张行业解决方案卡片及其链接保持不变。

## 实现范围

| 文件                                             | 变更                                                             |
| ------------------------------------------------ | ---------------------------------------------------------------- |
| `src/pages/index.astro`                          | 仅调整现有组件的输出顺序。                                       |
| `src/components/home/SolutionSplit.astro`        | 最小化调整章节和第一张卡片文案。                                 |
| `tests/rendered-site.test.ts` 或现有首页相关测试 | 断言首页章节顺序和关键业务文案，防止 AI 再次被放到具身智能之后。 |
| `说明文档.md`                                    | 记录本次信息架构决策、进度与验证结果。                           |

除上述文件外，不调整页面版式。若实施时发现必须修改样式文件，先停止并说明原因，请用户确认。

## 验收标准

- 首页 DOM 内容顺序为 AI 应用工厂、具身智能、合作伙伴、传统系统集成与行业场景。
- 合作伙伴轮播位于具身智能章节之后、传统系统集成章节之前。
- 首页 Hero、各章节现有布局和 CTA 链接保持不变。
- 375、768、1024、1440 宽度下阅读顺序正确、无横向溢出。
- `prefers-reduced-motion` 下内容顺序与可见性不变。
- 相关格式、类型、渲染与浏览器测试通过。

## 风险与处理

- 改变区块顺序可能使相邻背景色的视觉节奏不同。仅在产生明显断裂时调整现有区块的背景衔接，不改变其内部布局；如需更大范围视觉修改，先向用户确认。
- 文案调整不得改变已有路由含义。两个解决方案卡片继续链接到原有通用方案与行业方案入口。
